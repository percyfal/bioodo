bioodo.pandas module¶

bioodo.pandas.annotate_by_uri(func)¶

Decorator function annotate_by_uri.

Wrap function with wrapper that will optionally add an annotation to the data frame based on the uri. This can be particularly useful when merging several files and the data provenance needs to be tracked.

Example

The following example parses a qualimap output file and annotates the resulting data frame with the uri:

from bioodo import qualimap, odo
df = odo("Sample1/genome_results.txt", annotate=True)

The data frame will contain an extra column named “uri” with the value “Sample1/genome_results.txt”.

Alternatively, we can use a regular expression to parse the uri:

from bioodo import qualimap, odo
df = odo("Sample1/genome_results.txt",
         regex="(?P<sample>Sample[0-9]+)/(?P<file>.*)")

Here, the data frame will contain two extra columns, “sample” with value “Sample1”, and “file” with value “genome_results.txt”.

Parameters:
  • annotate (bool) – whether or not to annotate data frame
  • annotation_fn (function) – use custom function to annotate data frame
  • regex (str) – use regular expression to annotate data frame

bioodo.pandas.pivot(func)¶

Decorator function pivot.

Wrap function with wrapper that will pivot a data frame to wide format.

Example

The following example parses a qualimap output file and annotates the resulting data frame with a regex, at the same time pivoting the data frame so that the sample column is used to construct the new index.

from bioodo import qualimap, odo
df = odo("Sample1/genome_results.txt",
         regex=="(?P<sample>Sample[0-9]+)/(?P<file>.*)",
         index="sample",
         columns="statistic",
         values=["value"])

The arguments are identical to those in pandas.DataFrame.pivot().

Parameters:
  • index (str) – Column name to use to make new frame’s index. If None, uses existing index.
  • columns (str) – Column name to use to make new frame’s columns
  • values (str, list) – string or list. Column name to use for populating new frame’s values. If not specified, all remaining columns will be used and the result will have hierarchically indexed columns

bioodo.utils module¶

bioodo.utils.aggregate_factory(module)¶

Factory function to generate aggregation function.

Parameters:module (str) – module name
Returns:function

Given a list of input files for a bioinformatics application, seamlessly apply odo to each input file and concatenate the results. Additional file-based information, such as sample names, can be added on the fly by supplying a regular expression with group names that parses the file names. The regular expression group name will be added to the data frame.

Examples

The following example uses bioodo.qualimap.aggregate() to parse two qualimap output files, extract sample names via the regular expression, and concatenate the results together with an additional column named “sample”.

from bioodo import qualimap
qualimap_files = ["Sample1/genome_results.txt",
                  "Sample2/genome_results.txt"]
df = qualimap.aggregate(qualimap_files,
                        regex="(?P<sample>Sample[0-9]+)/genome_results.txt")
Parameters:
  • infiles (list) – list of input file names
  • regex (str) – regex pattern to parse file
  • parser (func) – bioodo parser function to use in case the generic parsing fails
  • outfile (str) – outfile name. Compression will be inferred from suffix
  • kwargs (dict) – keyword arguments
Returns:

Aggregated data frame or None if outfile given
